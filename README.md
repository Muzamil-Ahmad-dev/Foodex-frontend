 # Foodify Frontend

**Foodify Frontend** is the user-facing part of the Foodify platform, built with **React + Vite**. It provides a seamless, responsive, and interactive interface for browsing menus, placing orders, and tracking order history. Designed with **feature-driven architecture** and modular components, this project demonstrates senior-level frontend engineering practices.

---

## Project Overview

The frontend focuses on **user experience, responsiveness, and maintainability**:

- **Feature-based architecture:** each feature (menu browsing, cart, checkout, order history) is isolated for maintainability and scalability
- **Responsive design** optimized for desktops, tablets, and mobile devices
- **Smooth interactions and animations** using Framer Motion
- **Secure payment flow** integrated with Stripe (test mode)
- **Fast development** with Vite and React HMR

---

## Core Features

- Browse **food categories and menu items**
- Add items to **cart and checkout**
- View **order history**
- **Responsive and animated UI** with React, Tailwind CSS, and Framer Motion
- **Stripe integration** (test mode) for secure payments
- **Feature-based component structure** for maintainability

---

## Architecture & Engineering Highlights

- **Frontend:** React + Vite; feature-driven architecture for isolated and maintainable components
- **State Management:** Local component state and context for scalable data flow
- **UI/UX:** Tailwind CSS for responsive design and Framer Motion for smooth animations
- **Payment:** Stripe (test mode) for secure checkout flow
- **Deployment:** Hosted on Vercel for production-readiness
- **Code Quality:** ESLint rules applied for clean and consistent code
